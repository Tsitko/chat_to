## 📋 ТЕКУЩИЕ ЗАДАЧИ

### 1. Исправить переносы строк в нумерованных списках

**Проблема:**
В ответах LLM с нумерованными списками номер и текст отображаются на разных строках:
```
1.
текст элемента списка
```

Должно быть:
```
1. текст элемента списка
```

**Скриншоты:**
- См. файл `UI test/Чат.png` - показывает текущее состояние проблемы

**Уже предпринятые попытки:**
- Добавлен CSS: `li > * { display: inline; }`, `li p { display: inline !important; }`
- Добавлен кастомный компонент `li` в react-markdown для разворачивания `<p>` внутри `<li>`
- Проблема сохраняется несмотря на изменения

**Возможные причины:**
- react-markdown создает `<p>` внутри `<li>`, вызывая переносы
- CSS правила недостаточно агрессивны или перекрываются другими стилями
- Кастомный компонент li не работает корректно

**Следующие шаги:**
- Отладить кастомный компонент li в `AssistantMessage.tsx`
- Проверить, действительно ли `<p>` разворачиваются
- Рассмотреть альтернативный подход: использовать `rehype-raw` или другие плагины
- Проверить итоговый HTML в DevTools для диагностики

**Файлы для изменения:**
- `frontend/src/components/AssistantMessage.tsx` - кастомный компонент li
- `frontend/src/components/AssistantMessage.css` - CSS правила для списков
