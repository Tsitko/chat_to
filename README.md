# Chat To - Беседы с историческими личностями

Веб-приложение для разговоров с историческими персонажами на основе их книг с использованием RAG (Retrieval Augmented Generation) и локальных LLM.

## Основные возможности

- **Создание персонажей** - добавляйте исторических личностей с аватарами и книгами
- **Умные беседы** - общайтесь с персонажами, которые отвечают на основе своих произведений
- **Двойная база знаний** - система использует знания из книг и предыдущих бесед
- **Асинхронная индексация** - книги индексируются в фоне, не блокируя интерфейс
- **100% локально** - LLM работает через LM Studio, эмбеддинги через Ollama (без облака)
- **🎙️ Text-to-Speech (TTS)** - озвучивание сообщений персонажей с кнопкой воспроизведения
- **🎤 Speech-to-Text (STT)** - голосовой ввод сообщений с автоматической транскрибацией

## Технологический стек

**Backend:**
- Python 3.12+ с FastAPI
- SQLite + SQLAlchemy (async)
- ChromaDB для векторного поиска
- LM Studio для LLM, Ollama для embeddings

**Frontend:**
- React 18 + TypeScript
- Vite (сборка)
- Zustand (управление состоянием)
- React Dropzone (загрузка файлов)

**LLM Models:**
- `qwen/qwen3-30b-a3b-2507` - генерация ответов (LM Studio)
- `qwen-embeddings-indexer` - индексация документов
- `qwen-embeddings-kb` - поиск по базе знаний

## Быстрый старт

### Предварительные требования

1. **Python 3.12+** и **Node.js 18+**
2. **LM Studio** - запустите OpenAI-compatible API:
   - Host: `http://192.168.1.16:1234` (или задайте `LM_STUDIO_URL`)
   - Модель: `qwen/qwen3-30b-a3b-2507`

   ```bash

   ollama serve

   # Загрузка моделей (в отдельном терминале)
   ```

3. **TTS/STT сервис** (опционально, для озвучки и голосового ввода):
   - Запустите локальный сервис на порту 8013
   - Проект: `~/Projects/custom_tts`
   - Используется для синтеза речи и распознавания голоса

## Использование

### 1. Создание персонажа

1. Нажмите кнопку "+" в левой панели
2. Введите имя персонажа (например, "Гегель")
3. Загрузите аватар (PNG/JPG)
4. Загрузите книги (PDF/DOCX/TXT)
5. Нажмите "Создать"

Книги будут индексированы автоматически (это займёт время).

### 2. Начало беседы

1. Выберите персонажа из списка слева
2. Введите сообщение в поле внизу или используйте голосовой ввод (🎤)
3. Нажмите "Отправить" или Enter
4. Персонаж ответит на основе своих знаний из книг
5. Используйте кнопку 🔊 под сообщениями для прослушивания озвучки

### 3. Редактирование персонажа

1. Выберите персонажа
2. Нажмите кнопку редактирования в заголовке чата
3. Измените имя или добавьте новые книги
4. Сохраните изменения

### 4. Удаление

- **Персонаж**: в модальном окне редактирования есть кнопка удаления
- **Книга**: в списке книг персонажа можно удалить отдельные книги

### 5. Голосовые функции

**Text-to-Speech (TTS):**
- Кнопка 🔊 под каждым сообщением персонажа
- Клик → синтез речи → воспроизведение
- Повторный клик → остановка воспроизведения
- Кэширование аудио файлов в `backend/data/audio/`

**Speech-to-Text (STT):**
- Кнопка 🎤 рядом с кнопкой "Send"
- Клик → начало записи (отображается таймер)
- Повторный клик → остановка → транскрибация → автоматическая отправка текста

## Архитектура

### Как работает система

1. **Загрузка книг** → парсинг (PDF/DOCX/TXT) → разбиение на чанки → генерация embeddings → сохранение в ChromaDB
2. **Отправка сообщения** → поиск в базе знаний книг → поиск в базе знаний бесед → построение промпта → генерация ответа через LLM → сохранение в истории
3. **Индексация беседы** → сообщение пользователя индексируется в базу знаний бесед (асинхронно)

### Двойная база знаний

Каждый персонаж имеет **две независимые** базы знаний:

1. **Books KB** - индексированные книги (статичная база)
2. **Conversations KB** - история бесед (растущая база)

При ответе система ищет контекст в обеих базах.

## API Эндпоинты

### Персонажи
- `GET /api/characters` - список всех персонажей
- `POST /api/characters` - создать персонажа
- `GET /api/characters/{id}` - данные персонажа
- `PUT /api/characters/{id}` - обновить персонажа
- `DELETE /api/characters/{id}` - удалить персонажа
- `GET /api/characters/{id}/avatar` - получить аватар

### Книги
- `GET /api/characters/{id}/books` - список книг
- `POST /api/characters/{id}/books` - добавить книгу
- `DELETE /api/characters/{id}/books/{book_id}` - удалить книгу

### Сообщения
- `GET /api/characters/{id}/messages` - история сообщений (пагинация)
- `POST /api/characters/{id}/messages` - отправить сообщение

### Индексация
- `GET /api/characters/{id}/indexing-status` - статус индексации книг

## Troubleshooting

**LM Studio не отвечает:**
```bash
# Проверьте, что LM Studio запущен и API доступен
curl http://192.168.1.16:1234/v1/models
```

**Ollama не отвечает (embeddings):**
```bash
# Проверьте, что Ollama запущен
curl http://localhost:11434/api/tags

# Если нет, запустите
ollama serve
```

**Модели не загружены:**
```bash
# Проверьте список моделей
ollama list

# Загрузите нужные
ollama pull qwen-embeddings-indexer
ollama pull qwen-embeddings-kb
```

**Backend не запускается:**
- Проверьте, что виртуальное окружение активировано
- Проверьте, что все зависимости установлены: `pip install -r backend/requirements.txt`
- Проверьте логи в `backend.log`

**Frontend не запускается:**
- Проверьте Node.js версию: `node --version` (должно быть 18+)
- Переустановите зависимости: `cd frontend && rm -rf node_modules && npm install`
- Проверьте логи в `frontend.log`

**Индексация книг медленная:**
- Это нормально для больших файлов
- Проверьте статус: `GET /api/characters/{id}/indexing-status`
- ChromaDB индексация может занять несколько минут для книги на 200+ страниц
